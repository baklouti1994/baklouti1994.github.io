Outlier Detection in a Linear Signal with MCMC¶

Introduction :¶

In this notebook, we will simulate a linear model and then we will add some ouliers and try to detect them with the implementation of a Gibbs sampler.

MCMC :¶

Also known as the Markov Chain Monte Carlo is the main family of algorithms that uses the bayesian approach for statistical approach. It allows us to infere the posterior distribution. this family contains various algorithms such as Metropolis Hasting , Gibbs Sampler and also Hamiltonian Monte Carlo. In our case we will explain both of Metropolis Hasting and Gibbs Sampler algorithms and use The Gibbs Sampler to detect Outliers in a linear signal that we will simulate .

Why Do we use MCMC for Inference ? :¶

A statistician could say why do we use MCMC when we could just by using conjugate priors ( the posterior distributions $ \mathbb{p}(\theta | x ) $ are in the same probability family as the prior probability distribution $ \mathbb{p}(\theta)$ ) infere our posterior distributions . The answer to this question is that not all models can be defined by conjugate priors. There are may complicated models with numerous parameters (multi-dimensional) and the problem lies here : We can't with analytical method to infer posterior distributions , If this is the Bayes Rule :

$$ \mathbb{P}(\theta | D ) = \dfrac{\mathbb{P}(D | \theta ) \mathbb{P}(\theta)}{\mathbb{P}(D)} $$

Then We need to calculate $\mathbb{P}(D)$ , In order to do that , we need to evaluate this Integral : $$\int_{\theta} \mathbb{P}(D, \theta) d \theta $$

Due to the multi-dimensionality of the parameters we need to turn to a numerical approximation method instead and Monte Carlo seems to be the best candidate for the job.

The best thing about Monte Carlo Markov Chain is that they perform an intelligent search or scan the multi-dimensional space. As a matter of fact, the idea is to sample from the posterior distribution by combining Monte Carlo ( random search ) with the fact that those searches are memoryless (they don't depend on the initial state) which the basic idea of markov chains.

MCMC algorithms :¶

The Metropolis Hasting Algorithm :¶

Theory :¶

Metropolis-Hasting is a specific implementation of mcmc and it showed its performance with multi-dimentional data.

This algorithm requires a simple distribution function or a proposal distribution $\mathbb{Z}$ the point of this distribution is that M-H will used to randomly walk in the space and take values to accept them or reject them. this walk don't depend on the initial state ( The Markov Chain part of the algorithm).

The likelihood of each sample drawn is calculated with function that is proportional to the posterior that we want to take a sample from we'll call it $\mathbb{f}$ .

To get from a positon from a parameter $\theta$ to another we need just to propose a new parameter $\theta ' $ and draw a sample from the proposed disrtibution which is $ Q(\theta , \theta ') $ . And then to decide whether we accept this transition or not we need to compute this ratio : $$ \dfrac{\mathbb{P}(\theta '/ D ) }{ \mathbb{P}(\theta / D) } $$ Which is by usin the bayes formula :

$$ \dfrac{\mathbb{P}(D | \theta ' ) \mathbb{P}(\theta ')}{\mathbb{P}(D| \theta) \mathbb{P}(\theta )} $$

Algorithm :¶

1) Let $\theta^{(1)} $ be the initial sample from the distribution of $ (\theta) $

2) Sample $\theta ^* $ from $ Q(\theta , \theta ^*)$

3) Accept $ \theta ^*$ if : $$ \dfrac{ \mathbb{f}(D/ \Theta= \theta ') \mathbb{P}(\theta ')}{\mathbb{f}(D/ \Theta= \theta ) \mathbb{P}(\theta)} > r $$

With r = genetated a uniform random number in [0,1]

Gibbs Sampler¶

Theory :¶

The idea in Gibbs sampling is to generate posterior samples by using other variables : We sample from each variable' conditional variable's disribution with the other variables we'll explain with more details in the outlier detection script.

Algorithm¶

1) Initialize $x^{(0)}$ ~ $q(x)$ # prior distribution

2) for i = [1,n] do

$x_1^{(i)}$ ~ $\mathbb{p}(X_1=x_1/ X_2^{i-1}=x_2,...,X_D^{i-1}=x_d)$

$x_2^{(i)}$ ~ $\mathbb{p}(X_2=x_2/ X_1^{i}=x_1,X_3^{i-1}=x_3...,X_D^{i-1}=x_d )$

 .

 .

 .

$ x_D^{(i)}$ ~ $\mathbb{p}(X_D=x_d/ X_1^{i}=x_1,X_2^{i}=x_2...,X_{D-1}^{i}=x_{d-1}) $

  end for

Then , we write our regression model :

First of all , we choose the prior distribution for the slope , we'll use a modified Cauchy because it's an invariant distribution throught rotation of coordinates which is equivalent to a uniform density on the angle $\theta$ = arctan $\beta$ for more information please consult this article : https://projecteuclid.org/download/pdf_1/euclid.ba/1339616542

Then We define our Model as the deterministic part of the model.

In [51]:
@pymc.stochastic                                            # Decorated function in Python 
def slope_and_intercept(value=[1., 5.]):
    slope, intercept = value
      # uniform prior = arctan(slope)
         # d[arctan(x)]/dx = 1 / (1 + x^2)
    prob = np.log(1. / (1. + slope ** 2))     #loglikelihood 
    return prob

@pymc.deterministic
def model_(x=x, slope_and_intercept=slope_and_intercept):
    slope, intercept = slope_and_intercept
    fit = slope * x + intercept 
    return fit

we will reason on the inlier points , As a manner of fact , an inlier is represented by a Bernoulli : $$ p_{inlier} \sim \operatorname{Bern} \left({ 1 - p_{outlier}}\right) $$

Also, We define our posterior likelihood of the ourtlier so that we can detect them by having a low posterior.

def log_posterior_likelihood_of_outlier(y_with_outlier, mu, spread, inlier, mean_outliers, spread_outliers):
    inlier_posterior = np.sum(inlier * (np.log(2 * np.pi * spread ** 2) + (y_with_outlier - mu) ** 2 / (spread ** 2)))
    outlier_posterior = np.sum((1 - inlier) * (np.log(2 * np.pi * ((spread ** 2) + (spread_outliers ** 2))) + (y_with_outlier - mean_outliers) ** 2 / ((spread ** 2) + (spread_outliers ** 2))))
    return -0.5 * (inlier_posterior + outlier_posterior)

Then, we implement all of our models and variables into one model to estimate the parameters
